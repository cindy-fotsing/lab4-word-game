## First Impressions

# Initial Thoughts
My initial take was that the game logic seemed simple, but the strict constraints—specifically avoiding while True loops and string replacement—made the project significantly more complex than a standard Python assignment.

# Assumptions Made
I assumed that the game state would need to be passed through functions manually since the instructions emphasized pure functions and immutability.

# Points Needing Clarification
I initially needed clarification on how to handle user input without loops and how the "Agent Mode" in CoPilot differed from "Ask Mode" regarding journal updates.

## Key Learnings

# Computer Science Concepts and Technical Skills:
-Pure Functions: Learning to write functions that don't modify global state.
-Testing with Pytest: Setting up a virtual environment and writing automated tests for edge cases.
-Immutability: Handling data without changing the original list or string.

# Insights about Using CoPilot Effectively
-Using Ask Mode for logic brainstorming and Agent Mode for documentation and logging proved to be an efficient workflow.

# New Concepts or Tools Encountered
-Virtual Environments (venv): Isolating dependencies for the project.
-Pytest: A framework for professional-grade testing.

## Report on CoPilot Prompting Experience

# Types of prompts that worked well
-Specific "Adversary" prompts like "Can you suggest tests for this function?" were excellent for finding edge cases I missed.
